## 基线信息

- 产品名称：AgentSystem
- 产品类型：企业私有化多 Agent 代码协作平台
- 当前阶段：可运行本地 MVP，支持模拟和显式真实模型调用、持久化工作流、本地认证与 RBAC
- 目标阶段：单组织多仓库生产版，支持真实模型网关、隔离执行、人工审批、GitHub Enterprise 和完整审计
- 主要终端：桌面 Web；平板提供审批与查看能力；手机提供只读、通知和轻量审批能力
- 默认语言：简体中文，可切换英文
- 主题：浅色、深色、跟随系统
- 设计目标：任务优先、证据优先、安全可见、渐进披露

## 状态标记

文档中的能力使用以下状态：

- `Current`：当前代码已具备且可验证。
- `MVP`：下一生产切片必须完成。
- `Phase 2`：MVP 稳定后实现。
- `Future`：预留方向，不纳入当前承诺。
- `Simulated`：界面与记录存在，但不会触发真实外部调用。

## 需求变更规则

1. 需求变更先更新 `product-requirements.md`，保留需求编号。
2. 页面或流程变化同步更新 `ux-ui-design.md`。
3. 接口、数据或安全边界变化同步更新 `architecture.md`。
4. 进入开发前，将需求编号映射到 `development-plan.md` 中的里程碑和测试项。
5. 合并前必须满足对应验收标准、自动化测试和审计要求。

## 当前实现与目标设计的关系

当前实现保留为行为参考和可运行原型，不作为最终模块边界：

- `src/agentsystem/ui.py` 证明了工作台、任务、Agent、项目、模型、日志和设置的基本交互，但目标是拆分为独立前端应用和可复用组件。
- `WorkflowService` 证明了计划审批、测试修复、安全审查和 PR 审批的主流程，但目标是持久化、可恢复、幂等的异步工作流。
- `ModelGateway` 已提供每个 Agent 独立 provider/model、Keychain 凭据引用、真实适配器、模型发现和健康检查。
- `CapabilityRegistry` 已提供 MCP 工具发现/调用、Skill 解析、租户隔离和按 Agent 绑定。
- `ToolExecutor` 已提供权限边界，目标是迁移到隔离 workspace、受控命令模板和无公网默认策略。

## 术语

| 术语 | 定义 |
| --- | --- |
| Task | 用户发起的一次代码协作任务，是产品的核心业务对象 |
| Run | Task 的一次执行实例，可重试、恢复或重新运行 |
| Step | Run 中由 Agent、人工审批或系统执行的单个步骤 |
| Agent | 具有目标、模型路由、工具权限和 handoff 规则的执行角色 |
| Handoff | 一个 Agent 将上下文和控制权交给另一个 Agent |
| Artifact | 计划、补丁、测试报告、审查报告、PR 描述等可追溯产物 |
| Trace | 串联任务、模型调用、工具调用、审批和事件的可回放记录 |
| Credential Reference | 指向企业密钥管理系统中凭据的引用，不包含明文密钥 |
| Policy | 决定模型、工具、文件、网络和审批行为的可版本化规则 |
